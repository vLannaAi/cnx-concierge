Negative Application

Brand Application on Dark Backgrounds

Negative application refers to the brand presentation on dark backgrounds, specifically CNX Navy. This application is used sparingly for special applications such as hero sections, banners, and accent panels. All text must appear in white or light colors to maintain readability.

Usage Guidelines

Primary Use Cases

  • Hero sections on websites
  • Banner designs (vertical, horizontal, square)
  • Accent panels and callout sections
  • Special promotional materials
  • Event signage and displays

Best Practices

  • Use sparingly—negative application should be the exception, not the rule
  • Always use White text—never use colored text on dark backgrounds
  • Maintain sufficient contrast (minimum 4.5:1 for body text)
  • Use opacity variations (95% body, 80% secondary) for hierarchy
  • Reserve for high-impact applications where brand presence is critical
  • Never use negative application for long-form content

Contrast Requirements

  • Body text: White at 95% opacity meets WCAG AA standards
  • Headings: White at 100% opacity provides maximum contrast
  • Links: White with border ensures clear interactivity indication
  • Always test contrast ratios for accessibility compliance
